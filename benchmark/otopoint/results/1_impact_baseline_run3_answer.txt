## External Systems

| System | How It's Coupled to Order | What Breaks |
|--------|--------------------------|-------------|
| **PostgreSQL (Prisma)** | The Order table itself + FK constraints to `OrderItem`, `Payment`, `UserVoucher`, `ChatMessage`, `Printer` | Any field rename/removal requires migration; all `@generated` types must be regenerated with `prisma generate` |
| **Redis** | `OrderRedisService` publishes `{ orderId, merchantId, state, paymentStatus, estimatedTime, readyTime }` to channels `order:created:{merchantId}` and `order:updated:{orderId}`. Cached user/cart contexts in `PrinterRedisService` reference orderId | Payload shape changes break all active subscribers (mobile clients, CRM dashboards) |
| **Socket.io (WebSocket)** | `SocketService` emits `OrderUpdateEvent` and `OrderCreatedEvent` structs. Rooms: `order:{orderId}`, `merchant:{merchantId}:orders`, `{printerId}_client` | Event payload shape is consumed by mobile (`socket-lobby-consolidated.provider.tsx`) and CRM — any field change is a breaking API contract |
| **Stripe** | `PaymentService` uses `Order.total`, `Order.merchantId` to create PaymentIntent; `Payment.stripePaymentIntentId` links back. Stripe webhook must reconcile payment back to Order via `orderId` | Removing/renaming `total`, `merchantId`, or `paymentStatus` breaks Stripe charge flow and webhook reconciliation |
| **Centrifugo** | Chat messages (`ChatMessage.orderId`) and ORDER_PLACEHOLDER type reference Order. Centrifugo broadcasts chat events with order context | ORDER_PLACEHOLDER messages break if `orderId` FK is removed |
| **Bull Queues** | `VoucherQueueService` uses queued jobs that reference Order creation pipeline | Job payloads referencing removed fields will fail silently or throw at runtime |
| **FCM/Push Notifications** | `NotificationService` sends push with `orderId`, `state`, `merchantId` in payload | Notification payload will be malformed if fields are renamed |
| **Loyalty/Event Store (PostgreSQL)** | `PointsRewardService` creates `Transaction` events with `orderId` as event source reference | Breaks event sourcing audit trail if `orderId` is modified |
