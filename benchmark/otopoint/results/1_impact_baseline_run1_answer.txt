## External Systems

### Redis
- **Pub/Sub channels:** `order:created:{merchantId}`, `order:updated:{orderId}`, `order:cancelled:{orderId}` — published by `order.reddis.ts`; subscribed by `order.pubsub.ts`
- **Centrifugo retry queue:** `centrifugo:retry:queue` — stores failed Centrifugo publishes containing Order payloads
- **Printer/lobby context keys:** per-user printer state in Redis stores cart items that feed into `createOrderFromGroup()`; if Order fields change, the shape mismatch can corrupt order creation

### Socket.io (WebSocket)
- **Rooms:** `order:{orderId}` (customer tracking), `merchant:{merchantId}:orders` (dashboard)
- **Events emitted:** `order:created`, `order:updated`, `order:cancelled`, `order:subscribed`, `order:unsubscribed`, `Order.PURCHASED`
- **Client subscriptions:** `subscribe_order`, `unsubscribe_order`, `subscribe_all_orders` — all keyed on Order fields
- Adding/renaming fields changes the payload mobile/CRM clients receive; removing fields used by clients breaks real-time views

### Centrifugo
- Publishes order status events to `chat:{chatId}` channels
- `centrifugo.service.ts` serializes Order data into Centrifugo messages
- Retry mechanism (up to 3 attempts, exponential backoff) stores Order payloads in Redis queue

### Stripe
- `payment.service.ts` creates a `Payment` record with a `cartSnapshot` (cart items that become the Order)
- Stripe webhook → `fulfillOrder()` → `OrderService.createOrderFromCart()`
- `Payment.orderId` FK set after order creation; `Order.paymentStatus` updated on payment confirmation
- Changing `Order.total`, `Order.subtotal`, `Order.paymentMethod`, or `Order.paymentStatus` field names/types affects Stripe reconciliation logic

### Push Notifications (FCM/APNs via NestJS)
- `notification.service.ts` reads `Order.state`, `Order.userId`, `Order.printerId` to route and deduplicate notifications
- `UserNotification` and `NotificationLog` models store `orderId` references; removing the `id` field or changing the `state` enum values breaks notification delivery

### `@otopoint/pricing-engine` (external npm package)
- Consumes `OrderCalculationItem[]` — a projection of Order + OrderItem fields
- Any price-related field change (`subtotal`, `total`, `voucherDiscount`) requires coordinated package update
