## Events emitted

All calls are verified from source. Rooms and namespaces are explicit.

### Via `SocketService` high-level methods

| Event name | Method | Room(s) targeted | Emitting namespace(s) |
|---|---|---|---|
| `order:update` | `emitOrderUpdate` | `order:${orderId}`, `merchant:${merchantId}:orders`, `${printerId}_client` | mobile, CRM, legacy |
| `orders:update` | `emitOrderUpdate` | `merchant:${merchantId}:orders` | mobile, CRM, legacy |
| `orders:new` | `emitOrderCreated` | `merchant:${merchantId}:orders`, `${printerId}_client` | mobile, CRM, legacy |
| `voucher_update` | `emitToUser` / `emitVoucherUpdate` | `user:${userId}` | mobile, legacy |
| `lobby_voucher_update` | `emitToLobby` / `emitVoucherUpdate` | `lobby:${printerId}` | mobile, legacy |
| `merchant_voucher_update` | `emitToMerchant` / `emitVoucherUpdate` | `merchant:${merchantId}` | mobile, CRM |
| `menu_updated` | `emitMenuUpdated` | `printer:${printerId}` (mobile), `merchant_${printerId}` (CRM) | mobile, CRM |
| `points_claimed` | `emitToUser` | `user:${userId}` | mobile, legacy |
| `order:purchased` | `emitToUser` (`SOCKET_EVENTS.ORDER.PURCHASED`) | `user:${userId}` | mobile, legacy |
| `order_item:state_changed` | `emitToMerchant` | `merchant:${merchantId}` | mobile, CRM |
| `order_item:bulk_state_changed` | `emitToMerchant` | `merchant:${merchantId}` | mobile, CRM |

### Via `PrinterCommunicationService` (raw emit via SocketService servers)

| Event name | Room(s) targeted | Namespaces |
|---|---|---|
| `user-joined` | merchant room + `${printerId}_client` room | all 3 |
| `user-left` | merchant room + `${printerId}_client` room | all 3 |
| `voucher-activated` | merchant room | all 3 |
| `voucher-claimed` | `${printerId}_client` room | all 3 |
| `group-created` | `${printerId}_client` room | all 3 |
| `group-invite` | individual socket (found by user ID search) | mobile + legacy |
| `order-created` | merchant room | all 3 |

### Via `BaseSocket.emitStateChanged` (PrinterController / FactorySocket)
Emits `update_state` to the printer's state room via `socketService.server` (legacy namespace only in these callers).

## WebSocket Gateways

| File | Class | Namespace | Clients |
|---|---|---|---|
| `gateways/mobile.gateway.ts` | `MobileWebSocketGateway` | `/mobile` | React Native app |
| `gateways/crm.gateway.ts` | `CrmWebSocketGateway` | `/crm` | CRM web (Auth0 users) |
| `gateways/websocket.gateway.ts` | `WebSocketGateway` | `/` (default) | Legacy — redirects to the two namespaces above |

`SocketService` holds references to all three as `mobileServer`, `crmServer`, and `server` respectively. Each gateway sets its own field in `afterInit()`.
