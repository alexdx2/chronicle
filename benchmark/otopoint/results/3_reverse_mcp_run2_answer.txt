## WebSocket Gateways

| Gateway class | File | Namespace | Sets on SocketService | CRM event handlers |
|---|---|---|---|---|
| `WebSocketGateway` | `gateways/websocket.gateway.ts` | `/` (default, legacy) | `socketService.server` | — |
| `MobileWebSocketGateway` | `gateways/mobile.gateway.ts` | `/mobile` | `socketService.mobileServer` | — |
| `CrmWebSocketGateway` | `gateways/crm.gateway.ts` | `/crm` | `socketService.crmServer` | `subscribe_all_orders`, `unsubscribe_all_orders`, `subscribe_order`, `unsubscribe_order` |

All three gateways implement `OnGatewayConnection`, `OnGatewayInit`, `OnGatewayDisconnect`. Each gateway's `afterInit()` injects its `Server` instance into the corresponding property on `SocketService`, which is how `SocketService` can broadcast to namespace-specific rooms. Emit methods (`emitOrderUpdate`, `emitToUser`, etc.) iterate over all three server references.

## Downstream consumers

| Consumer | Channel | Events consumed |
|---|---|---|
| **Mobile app** (`/mobile` namespace) | Socket.io WS | `order:update`, `orders:new`, `order:purchased`, `points_claimed`, `voucher_update`, `lobby_voucher_update`, `menu_updated`, `user-joined` |
| **CRM web** (`/crm` namespace) | Socket.io WS | `order:update`, `orders:new`, `merchant_voucher_update`, `menu_updated` — also subscribes via `subscribe_all_orders` / `subscribe_order` server-side events |
| **Legacy clients** (default `/` namespace) | Socket.io WS | Same events as mobile; kept for backwards compat |
| **Bull queue workers** | Internal (Redis queue) | `voucher.bull.ts` processes voucher jobs and re-emits `voucher_update` / `points_claimed` via SocketService |
| **GraphQL subscriptions** | `order.pubsub.ts` bridges to GraphQL `PubSub` | `order:update` → GraphQL `ORDER_UPDATED` subscription (used by CRM Apollo subscriptions) |
