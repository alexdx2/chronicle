## WebSocket Gateways

Three gateways register themselves on `SocketService`'s server references during `afterInit()`:

| Gateway class | File | Namespace | `SocketService` property set |
|---|---|---|---|
| `WebSocketGateway` | `src/gateways/websocket.gateway.ts` | `/` (default, legacy) | `socketService.server` |
| `MobileWebSocketGateway` | `src/gateways/mobile.gateway.ts` | `/mobile` | `socketService.mobileServer` |
| `CrmWebSocketGateway` | `src/gateways/crm.gateway.ts` | `/crm` | `socketService.crmServer` |

All three gateways also **inject** `SocketService` themselves (to receive and store the `Server` instance). `SocketFactory` is also injected into the two client-facing gateways (`WebSocketGateway`, `MobileWebSocketGateway`) and calls `baseJoinPrinter()` on new connections.

## Downstream consumers

Who ultimately receives these events by client type:

| Consumer | Gateway / namespace | Events received |
|---|---|---|
| **Mobile app** (`/mobile` + legacy `/`) | `MobileWebSocketGateway` + `WebSocketGateway` | `order:update`, `orders:new`, `points_claimed`, `voucher_update`, `lobby_voucher_update`, `menu_updated`, `user-joined`, `user-left`, `voucher-activated`, `voucher-claimed`, `group-created`, `group-invite`, `order-created` |
| **CRM web** (`/crm`) | `CrmWebSocketGateway` | `order:update`, `orders:update`, `orders:new`, `merchant_voucher_update`, `menu_updated` (via `merchant_{printerId}` room), `order_item:state_changed`, `order_item:bulk_state_changed`, `user-joined`, `user-left`, `group-created` |
| **Customer web** (`/mobile` or legacy) | Same as mobile (anonymous users join via the same socket path) | `order:update`, `points_claimed` |

**Key architectural note:** `SocketService` is a singleton that holds three `Server` references (`server`, `mobileServer`, `crmServer`). Its named emit helpers (`emitOrderUpdate`, `emitVoucherUpdate`, etc.) fan out to all three namespaces in a single call. `PrinterCommunicationService` accesses the raw server refs directly (`.server?.to(room).emit(...)`) rather than using the helper methods, bypassing the fan-out abstraction.
