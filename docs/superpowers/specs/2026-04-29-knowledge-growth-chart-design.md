# Knowledge Growth Chart — Design Spec

**Date:** 2026-04-29
**Replaces:** Scan History flat list card on Overview tab

## Summary

Replace the "Scan History" card in the admin dashboard Overview tab with a stacked area chart showing node and edge count growth across scans. Each scan point is hoverable for details. No backend changes required.

## Data Source

- Existing endpoint: `GET /api/scans?domain=<key>`
- Returns `[]SnapshotRow` ordered newest-first
- Chart reverses to oldest-first for left-to-right timeline
- Fields used: `created_at`, `node_count`, `edge_count`, `revision_id`, `snapshot_kind`

## Visual Design

- **Chart type:** Stacked area chart with two series
- **Series:**
  - Nodes — solid line `#6b8f71`, gradient fill from 30% to 5% opacity
  - Edges — solid line `#b8860b`, gradient fill from 25% to 5% opacity
- **Axes:**
  - X-axis: scan timestamps, formatted via existing `formatTime()`
  - Y-axis: auto-scaled counts with horizontal dashed grid lines
- **Dots:** Each scan is a circle on both lines, radius ~3px
- **Legend:** Inline top-right, colored squares with labels
- **Height:** Fixed ~200px, width fills the card

## Interactions

- **Hover dot:** Tooltip appears showing revision ID, date, node count, edge count, snapshot kind
- **No click actions** — keeping scope minimal

## Empty State

- Fewer than 2 scans: show existing "No scans recorded" centered text

## Implementation

- Pure inline SVG generated by Alpine.js reactive data
- Replaces the current Scan History card (`index.html` lines 455-474)
- Same `<div class="card">` wrapper, new inner content
- Chart recalculates on `scans` data change
- Responsive: SVG viewBox scales to container width
- No new dependencies — vanilla SVG, matches existing dashboard patterns
- No backend changes — uses existing `/api/scans` response as-is

## Out of Scope

- Click-to-diff between revisions
- Toggle for change deltas (changed_node_count, changed_edge_count)
- Knowledge context branching visualization
- New API endpoints
